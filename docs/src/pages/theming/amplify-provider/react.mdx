### Amplify UI Design Tokens

Amplify UI Design tokens are the base of Amplifies Theming system. Through the design token system you can update and modify large portions of the application by updating general design tokens or you can target specific tokens that are applied to a single component.

## Color Mode

`AmplifyProvider` will accept a `colorMode` prop which can be used to activate styles reserved for specific color modes. The available color modes are `system`, `light`, and `dark` and the overrides for these color modes are declared as follows:

```jsx
const theme = {
  name: 'dark-mode-theme',
  overrides: [
    {
      colorMode: 'dark',
      tokens: {
        colors: {
          black: { value: '#fff' },
          white: { value: '#000' },
          overlay: {
            10: { value: 'hsla(0, 0%, 100%, 0.1)' },
            20: { value: 'hsla(0, 0%, 100%, 0.2)' },
            30: { value: 'hsla(0, 0%, 100%, 0.3)' },
            40: { value: 'hsla(0, 0%, 100%, 0.4)' },
            50: { value: 'hsla(0, 0%, 100%, 0.5)' },
            60: { value: 'hsla(0, 0%, 100%, 0.6)' },
            70: { value: 'hsla(0, 0%, 100%, 0.7)' },
            80: { value: 'hsla(0, 0%, 100%, 0.8)' },
            90: { value: 'hsla(0, 0%, 100%, 0.9)' },
          },
        },
      },
    },
  ],
};
```

For further details on using color modes see [Dark Mode](/theming/dark-mode)

## Nonce

`AmplifyProvider` accepts a `nonce` prop which is used to allow-list the generated inline style tags created by `AmplifyProvider`. The `nonce` prop should be filled on the server with the uniquely generated nonce value when the initial HTTP request is made for the page. For additional details on nonce values see the [MDN Documentation](https://developer.mozilla.org/en-US/docs/Web/HTML/Global_attributes/nonce)
